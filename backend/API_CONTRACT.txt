GUEST (no authentication required):

==> POST /api/v1/auth/signup
   - body:     { email: string, password: string, username: string }
   - response: { success: bool, message: string, user?: { id: string, username: string, email: string, role: string } }
   - sets http-only cookie with JWT (payload: { id, role })

==> POST /api/v1/auth/signin
   - body:     { email: string, password: string }
   - response: { success: bool, message: string, user?: { id: string, username: string, email: string, role: string } }
   - sets http-only cookie with JWT (payload: { id, role })


ALL ROLES (authenticated users + admin):

==> GET /api/v1/auth/me
   - response: { success: bool, user: { id: string, username: string, email: string, role: string } }
   - verifies JWT cookie + checks user still exists in DB

==> POST /api/v1/auth/logout
   - response: { success: bool, message: string }
   - clears the http-only JWT cookie

==> GET /api/v1/recent-books
   - response: {
       success: bool,
       books: [{ 
         id, title, description,
         cover_image_url,
       }, ...],
     }
   - Note: returns only the recent 10 books and NEVER returns pdf_url (security ofc)

==> GET /api/v1/books/:id
   - response: {
        success: bool,
        book: {
          id, title, genres: [{ id, name }, ...], cover_image_url, author, rating, price, description,
        }
   }
   - Note: NEVER returns pdf_url


USER (regular authenticated users):

==> POST /api/v1/orders/checkout
   - body:     { book_id: string }
   - response: { success: bool, checkout_url: string, chargily_payment_id: string }
   - Creates Chargily checkout session and returns the payment URL

==> POST /api/v1/orders/wishlist
   - body:     { book_id: string }
   - response: { success: bool, message: string } // adds the book to the orders table with status pending (wishlist), make the amount 0 and no chargily_payment_id since it's not a real order yet, just a wishlist item

==> GET /api/v1/me
   - response: {
       success: bool,
       profile: { id, username, email },
       orders: [{ id, book: { id, title, cover_image_url }, purchased_at, amount, status }, ...],
     }
   - Note: includes both orders and wishlist (pending) books for the user

==> GET /api/v1/books
   - query:    ?page=1&limit=10&search=keyword&genre_id=uuid
   - response: {
       success: bool,
       books: [{ 
         id, title, author, rating, price, description,
         cover_image_url,
         genres: [{ id: string, name: string }, ...]
       }, ...],
       pagination: { page, limit, total, totalPages }
     }
   - Note: NEVER returns pdf_url (security ofc)

==> GET /api/v1/books/:id/read
   - response: { success: bool, pdf_signed_url: string (valid 1 hour), title: string }
   - Checks that user has purchased the book → returns temporary signed URL (prolly S3, i will see later)


ADMIN ONLY:

==> GET /api/v1/admin/dashboard
   - response: {
       success: bool,
       data: {
         booksNum: number,
         usersNum: number,
         totalMoneyAmount: number,
         topSellingBooks: [{ id, title, author }, ...],
         latestOrders: [{ id, book: { id, title }, user: { id, username }, purchased_at, amount }, ...],
         recentReviews: [{ id, user: { id, username }, rating, content }, ...],
         numbersOfUsersPerMonth: [{ month: string, count: number }, ...] // for the last 3 months
       }
     }

==> GET /api/v1/admin/books
   - query: ?page=1&limit=20&search=keyword&genre_id=uuid
   - response: same as public /books but includes pdf_url for admin
   - Search behavior:
     * search is optional and case-insensitive (contains match)
     * search fields: title, author, description
     * empty/whitespace search means no filtering
     * pagination total/totalPages are calculated after applying search filter

==> POST /api/v1/admin/books
   - body: {
       title: string,
       author: string,
       rating?: number,
       price: number,
       description?: string,
       cover_image_url?: string,
       pdf_url: string,           - changed (UploadThing/S3 key)
       genresIds: string[]        - at least 1 genre
     }
   - response: {
       success: bool,
       message: string,
       book: { id, title, ... full book object }
      }

==> PUT /api/v1/admin/books/:id
   - body: {
       title?: string,
       author?: string,
       rating?: number,
       price?: number,
       description?: string,
       cover_image_url?: string,
       pdf_url?: string,
       genresIds?: string[]      - can update genres
     }
   - response: { success: bool, message: string, book?: { ... } }

==> DELETE /api/v1/admin/books/:id
   - response: { success: bool, message: string }

==> POST /api/v1/admin/genres
   - body: { name: string }
   - response: { success: bool, message: string, genre: { id: string, name: string } }

==> GET /api/v1/admin/genres
   - response: { success: bool, genres: [{ id, name }, ...] }

==> PUT /api/v1/admin/genres/:id
   - body: { name: string }
   - response: { success: bool, message: string, genre: { id, name } }

==> DELETE /api/v1/admin/genres/:id
   - response: { success: bool, message: string }

==> GET /api/v1/admin/users
   - query: ?page=1&limit=20&search=...
   - response: { success: bool, users: [{ id, username, email, status }, ...] }

==> PUT /api/v1/admin/users/:id
   - body: { status: 'activated' | 'disactivated' }
   - response: { success: bool, message: string, user?: { ... } } // change the status of the user to activated/disactivated
   - read the user id from the URL params and the status from the body, then update the user

==> PUT /api/v1/admin/settings
   - body: { email?: string, password?: string }
   - response: { success: bool, message: string, admin?: { ... } }

==> GET /api/v1/admin/orders
   - query: ?page=1&limit=20&search=...
   - response: {
       success: bool,
       orders: [{
         id, user: { id, username }, book: { id, title },  amount, status
       }, ...]
     }
   - Note: don't includes pending orders (wishlist), only cancelled/finished ones for admin review, do not return chargily_payment_id for security reasons
   - Search behavior:
     * search is optional and case-insensitive (contains match)
     * search fields: user.username, book.title, status
     * empty/whitespace search means no filtering
     * pagination total/totalPages are calculated after applying search filter

==> GET /api/v1/admin/reviews
   - query: ?page=1&limit=20&search=...
   - response: {
       success: bool,
       reviews: [{
         id, user: { id, username }, book: { id, title }, rating
       }, ...]
     }
   - Search behavior:
     * search is optional and case-insensitive (contains match)
     * search fields: user.username, book.title, content
     * empty/whitespace search means no filtering
     * pagination total/totalPages are calculated after applying search filter

==> GET /api/v1/admin/analytics
   - query: ?type=daily|weekly|monthly
   - response: {
       success: bool,
       data: {
         sales: [{ date: string, totalAmount: number }, ...],
         topSellingGenres: [{ genre: { id, name }, sales: number }, ...], // sales is the total amount of money made from books of this genre
         topSellingBooks: [{ book: { id, title }, sales: number }, ...],
         ordersCount: number, // total number of orders in the given period (depending on the type daily/weekly/monthly) and excluding pending/wishlist ones
       }
     }

WEBHOOK (public - no auth, must be CSRF attack exempt):

==> POST /api/v1/webhooks/chargily
   - body: Chargily webhook payload (checkout.paid, etc.)
   - response: { success: bool }
   - Validates signature → updates/creates matching Order record and sets status to finished/cancelled
